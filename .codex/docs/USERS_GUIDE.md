# 目的
この文書の目的は、トマト収穫シミュレータを使って、トマト収穫ロジックを開発、検証、改善するための利用方法を整理することである。

このプロジェクトでは、次の 2 つを達成対象とする。

1. トマト収穫シミュレータを構築する
2. そのシミュレータを使って、トマト収穫ロジックを完成させる

ここでいうトマト収穫シミュレータとは、単にロボットがトマトに近づくだけの可視化ではない。トマトを物理シミュレータ上で再現し、ロボットが一定の把持力で正しく把持したときにだけ枝から外れる状態を再現する環境である。正しく把持できない場合、トマトは重力により落下する。

この文書は、Isaac Sim の細かな操作説明書ではなく、ロボット開発エンジニアがこのシミュレータを使ってロジック検証を進めるための利用者ガイドとして扱う。

# 想定する利用者
- ロボット開発エンジニア
- MoveIt2 を使ったルールベース制御を試したい利用者
- OpenVLA などの学習ベースロジックを試したい利用者
- シミュレータ環境とロボットソフトを分離して開発、更新したい利用者

# このシミュレータの位置づけ
## 全体構成
- シミュレータ環境は `Isaac Sim`
- ロボットソフト側は `ROS 2`

## 責務分離
- `Isaac Sim` は、環境、ロボット、カメラ、トマト、枝、トレー、物理挙動を再現する
- `ROS 2` 側のロボットソフトは、認識、把持戦略、運動計画、制御ロジックを担う
- シミュレータ環境とロボットソフトは完全に分離する
- それぞれ独立してアップデート可能にする

この分離により、シミュレータ側を保ったまま `MoveIt2` ベースのロジックを差し替えたり、学習ベースの推論ノードへ切り替えたりできる。

# このシミュレータで再現すること
- トマト 1 個が、所定位置で枝についた初期状態
- ロボットがカメラでトマト位置を認識する過程
- ロボットハンドでトマトを把持する過程
- 一定の把持条件を満たしたときだけ、トマトが枝から取れる過程
- 正しく把持できない場合に、トマトが重力で落下する挙動
- 把持後にトマトをトレーへ搬送して配置するタスク

# 利用者がこのシミュレータで試すこと
- `MoveIt2` を用いたルールベースの収穫ロジック
- `OpenVLA` などの学習ベースロジック
- 認識結果から grasp pose を生成するロジック
- 把持失敗時、落下時、再試行時の挙動確認
- シミュレータ環境を変えずにロボットソフトだけ更新する試験

# 利用前に知っておくこと
## 最小用語
- `3DView`
  - シミュレータ空間を表示する画面である
- `Fixed Camera`
  - ロボットとトマト全体を俯瞰する固定外部視点である
- `Hand Camera`
  - ロボットハンドに取り付けた視点である
- `Start`
  - ロボットソフトを開始し、収穫シナリオを走らせる
- `Stop`
  - 実行中のロボットソフトを停止する
- `Reset`
  - シミュレータ環境を初期状態に戻す

## 初期状態
- トマトは 1 個だけ存在する
- トマトは枝についた状態で初期配置される
- ロボットはホーム姿勢から開始する
- 利用者は 3DView 上の control window から開始、停止、初期化を行う

# 起動方法
## 通常の起動
```bash
./build.sh
./run.sh
```

`./run.sh` を実行すると、Isaac Sim のネイティブウィンドウが開き、その中に `3DView` と `Tomato Harvest Controls` ウィンドウが表示される。

Linux の X11 環境では、必要に応じて先に次を実行する。

```bash
xhost +local:root
```

## デバッグコンテナに入る
```bash
./into.sh
```

## コンテナ内からシミュレータを起動する
```bash
/isaac-sim/python.sh scripts/run_poc.py --mode isaac
```

## headless で確認する
```bash
/isaac-sim/python.sh scripts/run_poc.py --mode isaac --headless --test
```

# 起動後に何が見えるか
起動に成功すると、利用者は次を確認できる。

- Isaac Sim の `3DView`
- `Tomato Harvest Controls` ウィンドウ
  - `Start`
  - `Stop`
  - `Reset`
  - `Fixed Camera`
  - `Hand Camera`
- ターミナルログ
  - 現在フェーズ
  - 認識結果
  - トマトの camera 座標
  - トマトの world 座標
  - 成功、失敗理由

# 操作方法
## 1. 起動する
利用者は `./run.sh` を実行する。

期待する結果:
- Isaac Sim ウィンドウが開く
- 3DView が表示される
- `Tomato Harvest Controls` が表示される
- 状態表示が `Ready` になる

## 2. シーンを確認する
最初は `Fixed Camera` で全体を見る。

期待する結果:
- Franka Panda が見える
- 枝についたトマト 1 個が見える
- トレーが見える
- トマトが開始直後に落下していない

## 3. カメラ視点を切り替える
`Fixed Camera` と `Hand Camera` を切り替える。

期待する結果:
- `Fixed Camera` では全体配置が分かる
- `Hand Camera` ではロボットハンド搭載カメラの視点を確認できる

使い分け:
- 収穫タスク全体を見たいときは `Fixed Camera`
- 認識と接近の見え方を見たいときは `Hand Camera`

## 4. Start を押す
`Start` を押すと、ロボットソフトが開始され、収穫シナリオが進む。

シナリオは次の流れで進む。

1. トマト位置を認識する
2. ハンドをトマトへ接近させる
3. トマトを把持する
4. 把持できた場合だけ枝から外す
5. トレーへ搬送して配置する

期待する結果:
- ロボットがトマトを認識する
- ターミナルにトマト座標が出る
- ハンドがトマトへ移動する
- 正しく把持できれば、トマトが枝から外れる
- 最後にトレーへ搬送される

## 5. Stop を押す
`Stop` を押すと、実行中のロボットソフトを停止する。

期待する結果:
- 現在のシナリオ進行が止まる
- 状態表示が `Stopped` になる

## 6. Reset を押す
`Reset` を押すと、シミュレータ環境を初期状態へ戻す。

期待する結果:
- ロボットがホーム姿勢へ戻る
- トマトが枝についた初期位置へ戻る
- 視点が `Fixed Camera` に戻る
- 状態表示が `Ready` になる

重要:
- `Start` はシミュレーション開始である
- `Reset` は環境初期化である
- トマトを初期位置へ戻すのは `Reset` の役割である

# 利用者が確認すべき成功条件
- 起動後、トマトが枝についたまま保持されている
- `Start` でロボットソフトが動き出す
- トマト認識結果がログに出る
- 把持成功時だけ、トマトが枝から外れる
- 把持失敗時は、トマトが重力で落下する
- `Reset` でトマト、ロボット、視点が初期状態へ戻る
- 同じ環境のままロボットソフトを再試行できる

# このシミュレータの使い方の考え方
## シミュレータ環境側で扱うもの
- トマト、枝、トレーなどの scene
- カメラ
- 物理挙動
- Start / Stop / Reset の最小 UI

## ロボットソフト側で扱うもの
- トマトの認識
- grasp pose 生成
- 経路計画
- 把持戦略
- 搬送戦略

## 期待する開発フロー
1. シミュレータ環境を固定する
2. ROS 2 側で収穫ロジックを実装する
3. 3DView 上で Start して挙動を確認する
4. 失敗ケースを観察し、ロジック側を改善する
5. 必要に応じてシミュレータ環境だけ、またはロジックだけを独立更新する

# 現時点の PoC と本番の境界
- いまの PoC は、`シミュレータ環境とロボットロジックを分離して開発できる` ことの検証が主目的である
- 物理挙動は導入済みだが、fruit-stem の本番向け breakable joint 設計は今後の課題である
- ロボット接近と把持の細かな詰めは、PoC の手書き IK ではなく、今後 `MoveIt2` や学習ベースロジックで改善する前提である
- したがって、この PoC では `ロジック検証が回せる土台` を優先し、最終的な収穫性能の完成は次フェーズで扱う

# 注意
- 起動直後は shader warm-up で重いことがある
- `Hand Camera` は近接視点なので、全体確認には向かない
- GUI が開かない場合は、`DISPLAY` と X11/VNC セッションを確認する
- PoC 時点では、物理保持と把持判定の安定化のために簡略化した実装が含まれる
