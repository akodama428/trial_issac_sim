# Architecture Design

## 目的
`docs/ADR.md` で決定されたアーキテクチャを前提に、`docs/ARCHITECTURE.md` へ落とし込むための詳細アーキテクチャを設計する。

## 先に読むもの
- `AGENTS.md`
- `docs/MASTER.md`
- `docs/PROJECT.md`
- `docs/ADR.md`
- `docs/ARCHITECTURE.md`

詳細アーキテクチャ設計は、必ず `docs/PROJECT.md` の要件と `docs/ADR.md` の決定内容に基づいて行う。

## 手順
1. `PROJECT.md` の要件と `ADR.md` の決定内容を読む。
2. `ADR.md` に推奨案、主要コンポーネント、要件番号とモジュールの対応表があることを確認する。
3. `ADR.md` に不足があり詳細化できない場合は、その不足を明示し、必要なら `ADR.md` の更新または更新提案を行う。
4. まずシステム全体のアーキテクチャ図を作成する。
5. 次に、各主要モジュールについて詳細アーキテクチャを決める。
6. 最後に、要件 ID とモジュールの機能割付表を出力する。
7. 結果を `ARCHITECTURE.md` に記録する。

## 各段階で決めること

### 1. アーキテクチャ図の作成
少なくとも次を明確にする。
- 主要モジュール
- モジュール間の依存方向
- 主なリクエストフローまたはデータフロー
- 外部境界
- 永続化や成果物保存の位置

図は、少なくとも 1 つは全体像が分かる形式で示す。

### 2. 各モジュールの詳細アーキテクチャ
各モジュールについて少なくとも次を決める。
- モジュールの責務
- 入力
- 出力
- 内部で保持する主要データ
- 依存先
- エラー時の振る舞い
- 他モジュールとのインターフェース

### 3. 要件 ID とモジュールの機能割付表
`PROJECT.md` の要件番号を基準に、各要件をどのモジュールが主担当として実現するかを明示する。

## 必須出力
`ARCHITECTURE.md` には少なくとも次を含める。
- システム構成
- アーキテクチャ図
- 主要コンポーネント
- 各モジュールの詳細アーキテクチャ
- 処理フロー
- 設計上の前提
- 要件 ID とモジュールの機能割付表

## クリーンアーキテクチャ確認項目

### 単一責務の原則
各主要モジュールが、ひとつの主たる変更理由だけを持つか確認する。

確認例:
- UI モジュールが Git 操作まで持たないこと
- 成果物保存モジュールがカテゴリ分類ロジックまで持たないこと
- アーキテクチャ抽出モジュールが UI 表示都合に依存しないこと

### 依存関係ルール
依存関係が、より安定したポリシー層へ inward に向かっているか確認する。

最低限の確認項目:
- 中核のビジネスルールが UI に依存していない
- 中核のビジネスルールが DB、Git、外部 API の実装詳細に直接依存していない
- インフラが中核の契約に依存しており、その逆になっていない
- ビジネスルールが、アプリ操作、UI、DB、外部 API から分離されている

## やってはいけないこと
- `ADR.md` にないアーキテクチャ判断を勝手に既成事実化すること。
- `PROJECT.md` にない要件を推測して詳細設計へ埋め込むこと。
- 役割が曖昧なままモジュールを並べること。

不足している要件や ADR 上の判断がある場合は、それを明示したうえで、ユーザーに提案すること。

## ARCHITECTURE.md の推奨構成

```md
# ARCHITECTURE.md

## システム構成
<全体像の説明>

## アーキテクチャ図
<図または図の説明>

## 主要コンポーネント
### <モジュール名>
- 責務:
- 入力:
- 出力:
- 依存先:
- エラー時の振る舞い:

## 処理フロー
<主要ユースケースごとの流れ>

## 設計上の前提
<前提、制約、境界条件>

## Requirement to Module Mapping
| 要件番号 | 要件概要 | 主担当モジュール | 備考 |
| --- | --- | --- | --- |
| REQ-xxx | <要件の要約> | <モジュール名> | <必要なら記載> |
```

## リポジトリ固有の指針
- `ADR.md` で決定済みの推奨案を前提に詳細化する。
- `PROJECT.md`、`ADR.md`、`ARCHITECTURE.md` の役割分離を維持する。
- 詳細アーキテクチャは `ARCHITECTURE.md` に出力し、ADR の再比較は混ぜない。
